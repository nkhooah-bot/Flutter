Installing & Setting up Flutter and Android Studio

Lecture 2 Environment Initialization Guide
0 / 6 steps Setup Progress
1
Download and Extract the Flutter SDK
▼

Get the official Flutter SDK build asset package matching your local machine architecture:

  1. Go to the official Flutter download hub and obtain the stable distribution zip bundle.
  2. Create an installation sandbox directory path (e.g., C:\src\flutter or ~/development/flutter).
Warning: Do NOT target system directories requiring explicit higher authorization policies such as C:\Program Files\.
2
Configure Global Path Environment Variables
▼

Append the executable binaries context path lookup to local workspace system configuration parameters:

For Windows Systems: Search for "Edit the system environment variables", enter environment definitions, and append the full absolute path of your flutter\bin terminal module directly into the Path entry array block.

For macOS / Linux Systems: Append the export directive explicitly directly inside your shell configuration profile script file (~/.zshrc or ~/.bashrc):

export PATH="$PATH:$HOME/development/flutter/bin"
3
Install Android Studio IDE Application
▼

Configure the foundational background engine components for compiling target Android code packages:

  • Download and execute the official desktop installation setup client for Android Studio.
  • Follow the system setup wizard directives carefully and confirm the installation of both the Android SDK Engine and necessary Platform Tools.
4
Accept Android Command Line Tool Licensing
▼

Open Android Studio, select the SDK Manager settings block, choose SDK Tools tab, verify that Android SDK Command-line Tools (latest) is activated, and run the following in your terminal application to approve ecosystem compliance agreements:

$ flutter doctor --android-licenses

Accept each of the sequentially output license agreement validation checks by entering y when prompted by the terminal engine.

5
Set Up Your Target Android Emulator Device
▼

Configure a simulated device instance directly inside your workspace framework environments for execution tests:

  1. Select the Virtual Device Manager utility inside your IDE setup.
  2. Click Create Device, choose a device profile layout configuration (e.g., Pixel 7), download a target system runtime image version (e.g., API 34), and complete the profile instantiation steps.
6
Verify Overall System Integrity Checks via Doctor
▼

Execute the analytical check diagnostic operation framework to confirm environment variables are successfully mapped:

$ flutter doctor

Verify that the terminal console parameters report no warning blocks. Once all dependencies show verified status checkmarks, your workspace configuration setup is complete.
