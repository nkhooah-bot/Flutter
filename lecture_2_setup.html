Installing & Setting up Flutter and Visual Studio Code

Lecture 2B Environment Initialization Guide
0 / 6 steps Setup Progress
1
Download and Extract the Flutter SDK
▼

Get the official Flutter SDK build asset package matching your operating system configuration:

  1. Navigate to the official Flutter download hub and obtain the latest stable distribution zip bundle.
  2. Create a dedicated workspace installation directory (e.g., C:\src\flutter on Windows or ~/development/flutter on macOS/Linux).
  3. Extract the contents of the zip archive directly into that destination path.
Warning: Do NOT extract the SDK into system directories requiring administrative access elevations, such as C:\Program Files\.
2
Configure Global System Environment Variables
▼

Expose Flutter executable binaries globally so terminal instances can locate them automatically:

For Windows Systems: Search for "Edit the system environment variables" in your start menu, click "Environment Variables", locate the Path entry under user variables, click Edit, and append the absolute path to your flutter\bin directory.

For macOS / Linux Systems: Append the export path directive directly into your shell context configuration profile script (such as ~/.zshrc or ~/.bashrc):

export PATH="$PATH:$HOME/development/flutter/bin"
3
Install Visual Studio Code IDE Application
▼

Set up your primary lightweight code editing interface and environment engine:

  • Download the official installer binary from the code.visualstudio.com portal.
  • Run the installer and follow the setup wizard directives. On Windows, make sure to check the option to "Add to PATH" to enable terminal command line tracking.
4
Install Dart & Flutter Extensions in VS Code
▼

Equip your editor workspace with full autocomplete compilation tooling, syntax helpers, and debugger integrations:

  1. Launch Visual Studio Code and click the Extensions Marketplace icon on the left Activity Bar (or press Ctrl+Shift+X / Cmd+Shift+X).
  2. Search for "Flutter" (published by flutter.dev) and click Install. This action will automatically discover and install the companion Dart extension asset.
  3. Highly Recommended Optional Extension: Search for and install "Error Lens" to display warning logs and compilation problems instantly inline inside your code canvas lines.
5
Configure Android Toolchains & Software Licences
▼

Even when using VS Code as your main IDE editor, Flutter requires core background utilities provided by the Android SDK framework for device deployments:

  • Ensure Android Studio or the standalone Android SDK command-line options bundle is present on your system with Android SDK Command-line Tools (latest) activated.
  • Open a new system command prompt or shell window and execute the global compliance validation prompt to sign agreements:
$ flutter doctor --android-licenses

Type y (yes) to accept each sequential ecosystem license agreement check output by the platform engine.

6
Execute Environment Diagnostic System Checks
▼

Run the analytical diagnostic engine package to verify all environmental paths and editor dependencies map successfully:

$ flutter doctor

Verify that both the VS Code entry and Android Toolchain check blocks show valid green verification marks. Once confirmed, your localized coding workstation setup is ready for project creation.
